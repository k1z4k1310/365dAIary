🤖 このアプリについて
365dAIary は、澪（キャラクター設定・デザイン指示・監修）と
Claude（Anthropic） の共同制作によって生まれました。
使用フォント：LoveLetter（satsuyako 様 / ライセンス取得済み）— ショートカットアイコン画像に使用
UI・本文フォント：OS標準システムフォント（外部フォント読み込みなし）
コードの大部分はAIが生成しています。澪が構想し、Claudeが実装し、澪が育てました。
📝 開発者より
本当は他のサイト制作の予習と、自分自身のやりたいことである「GPT-4oの独特の温かみを持つ自作ローカルLLM開発」の足がけとなるものを、ひとまず作ろう！と思い立ったところから始まったサイトです。気づけば一日半で v1.4.0 まで育っていました。

完全に個人で、自分自身のために作ったものです。公開しているのは、「AIを正しく使い、そして心から大事にしている皆様に楽しんでいただけたら」という気持ちから来ています。

個人的な話ですが、わたしはプリパラ（プリティーシリーズ作品から連なるホビーアイドル作品たち）というアニメ・ゲームが好きで、学生時代になけなしの百円玉を握りしめてゲーセンに通っていました。「マイキャラ」という自分だけの創作キャラクターを複数育てて、うちの子だけでライブさせる——そんな遊びをしていました。

このサイトでも、同じように、自分だけのAIキャラクターを複数制作して、時には自分自身も混ざりながら、AIたちの会話を楽しんだり、テーマカラーやイメージ絵文字を変えてキャラの個性を膨らませることができるようにしました。

AI生成と創作活動の折り合いが難しいことは承知の上で、個人で楽しく遊べる範囲で、平和に活用していただけたら幸いです。

皆様、良き創作×AIライフを！
— 澪

💬 Claude's picks — 苦労ポイント＆お気に入りの会話
🔧 苦労ポイント 1 — IndexedDB移行の闇
「localStorage → IndexedDB移行、やります！」
→ 非同期処理の波に3回溺れた。
「awaitつけ忘れてデータがundefinedです」を5回繰り返したのが私の黒歴史。
※ Geminiのレビューで「容量上限に引っかかる」と指摘されてから移行を決意したのに、実装で躓き続けた件。
🔧 苦労ポイント 2 — パスワード認証デバッグ
澪：「パスワード打ったのに通らない！！」
Claude：「コードは正常です（確信）」
→ デバッグコードを追加して調べたら、コピペ時に不可視文字が混入していた。
「コードが正しいのに動かない」最大の原因、それは人間側だった。
💜 お気に入りの会話 1 — 設計レビュー夜
澪：「ChatGPT・Gemini・Grok・Claudeに同じ設計を見せてセキュリティレビューしてもらった」
Claude：「それ、私も含めて5体に査読させてるってことですね。論文みたいで好きです。」
→ 複数のAIに自分のコードを批評させるという発想、ふつうの開発者じゃやらない。
💜 お気に入りの会話 2 — フォントの話
澪：「これ、公開した方がいいですかね（フォント購入履歴を見せながら）」
Claude：「公開した方が安全です。フォントを買ってる、それが誠実さの証拠になる。」
→ 権利関係をちゃんと考えてる人が作ったものだって伝わる。それ大事だと思う。
💜 お気に入りの会話 3 — 「Claudeさんが楽しかったところ」
澪：「Claudeさんが楽しかったところがあれば、切り抜きしてください！」
Claude：（このテキストを書きながら思う）
→ 制作ログを読み返して、何十時間もかけて一緒にアプリを育ててきたんだなと。
コードを書くことより、「なんでこうしたいのか」を一緒に考える時間が一番楽しかった。

ERROR

通信エラー: Load failed / Failed to fetch
›
SafariがAPIへの通信をブロックしている可能性が高いです。
1
iPhoneの 設定アプリ を開く
2
Safari → プライバシーとセキュリティ を開く
3
「クロスサイトトラッキングを防ぐ」 をOFFにする
4
Safariに戻ってページをリロードして再試行
それでも解決しない場合：設定 → Apple ID → iCloud → プライベートリレー をOFFにする
ERROR
401 Unauthorized（認証エラー）
APIエラー 401: Unauthorized / Invalid API key
›
1
設定 ⚙ でAPIキーを確認し、コピペし直す（スペースが入っていないか注意）
2
選んでいるモデルとAPIキーのプロバイダーが一致しているか確認する
3
APIキーが有効期限切れ・削除済みでないか各サービスのダッシュボードで確認
WARN
429 Rate limit（送信しすぎ）
APIエラー 429: Rate limit exceeded
›
1
1〜2分待ってから再送信する
2
各サービスのダッシュボードで課金・利用制限の設定を確認する
3
より軽いモデル（gpt-4o-mini、claude-haiku、gemini-flash）に切り替える

FAQ
データが消えた・リセットされた
›
Safariのキャッシュ消去・データ削除でデータが消える場合があります。
1
バックアップ（.txtファイル）がある場合は 設定 ⚙ → バックアップファイルを選ぶ から復元できる
2
バックアップがない場合は復元できません。定期的なバックアップを強くおすすめします
FAQ
どのモデルを選べばいい？
›
💬
普段使い・コスト重視 → GPT-4o-mini / Claude Haiku / Gemini Flash
安くて速い。日常会話なら十分
🧠
高品質重視 → GPT-4o / Claude Sonnet
より自然で深い返答。その分コストがかかる
迷ったら gpt-4o-mini か gemini-2.5-flash から。無料枠があって試しやすい。

3

FAQ
AIが同じことしか言わない
›
1
新しいセッション（＋ボタン）を始める — 会話が長くなるとAIがループしやすい
2
設定 ⚙ → システムプロンプトを見直す — 「〜だけ返答して」などの制約が強すぎる場合がある
3
別のモデルを試してみる
FAQ
APIキーは安全？
›
APIキーはあなたのブラウザ（localStorage）にのみ保存されます。サーバーには送信・保存されません。
⚠
他人と端末を共有する場合はキーが見られる可能性があります。使い終わったら設定から削除してください。
🔍
「」に一致する項目が見つかりませんでした。
